How json and xml data is formed - JSON (JavaScript Object Notation) and XML (eXtensible Markup Language) are two formats commonly used to structure and transfer data. Here's a breakdown of how data is formed in each format: Readability: JSON is generally considered more readable and concise compared to XML. Data Types: JSON supports basic data types (strings, numbers, arrays, objects, booleans, null) natively. XML data is represented as text, and data types need to be interpreted. Structure: JSON is more compact with its key/value pair structure, whereas XML’s nested elements can result in more verbose data. Usage: JSON is widely used in web APIs and configuration files, while XML is often used in document formats, configuration files, and in applications where complex data representation is necessary.\
